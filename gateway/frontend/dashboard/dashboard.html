Guía Visual de la App

Base oficial obligatoria: patrones PMO + ERP. Todo módulo nuevo o editado debe respetar esta línea visual, incluyendo contenedor principal sin cuadro de fondo.

Fuente Oficial PMO + ERP

PMO y ERP son la referencia canónica de UX para MONSTRUO. No se aceptan variantes visuales paralelas.

Layout`section-block` + `section-header` (sin cuadro base)
Navegación`tab-bar` + `tab-btn`
Datos`monstruo-table` / `erp-table`
Acción`btn-primary` neon + dark surface

Contrato de Componentes

La composición visual mínima debe ser uniforme para que cualquier agente pueda intervenir sin romper consistencia.

Panel principal Subsecciones Acciones arriba 
div.main-inner.module-shell
  div.section-header.module-tabs-header.module-shell-header
  div.tab-bar (si aplica)
  section.section-block.module-shell-content

Checklist para Agentes

Checklist obligatorio antes de cerrar un cambio de UI en cualquier módulo.

1.Paleta PMO/ERP (`--neon`, `--info`, `--danger`)
2.Sin cajas blancas o inputs nativos sin tema
3.Contenedor principal sin cuadro (transparente)
4.Tablas/modales en superficie oscura consistente
5.Evitar inline style salvo excepción justificada
6.No romper contratos públicos del módulo
