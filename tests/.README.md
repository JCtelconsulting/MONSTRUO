# Tests
Validaciones automatizadas y verificaciones de hardening del proyecto.

## Estructura Permitida (Allowlist)
La IA NO debe crear archivos fuera de esta lista sin permiso.
- .README.md
- _helpers.py
- e2e_api_full.py
- e2e_ticketera.py
- verify_hardening.py
- __pycache__/  # runtime, no versionar contenido

## Politica Operativa
- No hardcodear credenciales en scripts.
- Usar `MONSTRUO_TEST_BASE_URL`, `MONSTRUO_TEST_USER`, `MONSTRUO_TEST_PASSWORD`.
- Bloquear ejecucion en URLs que parezcan PROD salvo `--allow-prod`.
- Mantener cada script con salida clara `PASS/FAIL` y codigo de salida no-cero al fallar.

## Ejecucion Recomendada
```bash
python3 tests/verify_hardening.py
python3 tests/e2e_api_full.py
python3 tests/e2e_ticketera.py
```
